**使用者：**

**助理回應：**

### 更新亮點：
1.  **分步數據採集**：後端 `app.py` 的 `run_vi` 路由現在會完整記錄每一次迭代的 `Value` 與 `Policy` 狀態，並將其作為歷史紀錄（History）一次性回傳給前端。
2.  **動態動畫渲染**：
    *   前端 JavaScript 使用了 `Promise` 與 `setTimeout` 實作非同步延遲。
    *   點擊按鈕後，網頁會以 **0.5 秒為間隔**，逐一顯示每一代的計算結果。
    *   按鈕文字會實時顯示當前的迭代步數（例如：「迭代第 12 步...」）。
3.  **最终路徑高亮**：當所有迭代完成且收斂後，系統會自動在 Value Matrix 與 Policy Matrix 上同步高亮顯示 **黃色的最佳路徑**。
4.  **視覺回饋**：迭代過程中按鈕會進入禁用狀態（Disabled），直到動畫播放完畢，防止重複觸發。

### 如何查看效果：
1.  開啟網頁 [http://127.0.0.1:5000](http://127.0.0.1:5000)。
2.  點擊 **「開始價值迭代」**。
3.  您將看到網格中的數字與箭頭從初始狀態開始，緩慢且規律地演變，直到最終收斂並畫出黃色路徑。

如果您覺得 0.5 秒太快或太慢，可以在 `index.html` 的 `await sleep(500)` 處調整數值（單位為毫秒）！
